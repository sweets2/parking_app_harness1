## F-17.5 Street Popup Click Wiring `[TODO]`

`app/main.ts` — wires the map click handler to show street cleaning info when the app is in parked mode, or set the tapped position when in browsing mode. Extends the `registerMapClickHandler` callback set up in F-10.

Wiring steps required in `main.ts`:
1. Import `showStreetPopup` from `./map` and `getStreetName` from `./geo`.
2. Import `StreetCleaningEntry` type from `../shared/types`.
3. Declare a module-level `cleaningEntries: StreetCleaningEntry[]` variable (initially empty).
4. After `initMap()`, fetch `data/street-cleaning.json` and populate `cleaningEntries`. Failure is non-fatal — catch and leave the array empty.
5. Add a `normalizeStreet(s: string): string` helper that lowercases and expands common abbreviations (St→street, Ave→avenue, Blvd→boulevard, Dr→drive, Pl→place, Hwy→highway).
6. Add a `findCleaningEntries(roadName: string): StreetCleaningEntry[]` helper that filters `cleaningEntries` by normalized street name equality.
7. In the `registerMapClickHandler` callback, branch on app mode:
   - **browsing mode**: call `renderPositionMarker(lat, lng)` and store the tapped coordinates as the current position (existing F-10 behavior — do not duplicate, extend it).
   - **parked mode**: call `getStreetName(lat, lng)`, then `showStreetPopup(lat, lng, road, findCleaningEntries(road))` if road is non-null.

Note: `data/street-cleaning.json` is served from `app/data/street-cleaning.json` — the build step (`npm run build`) copies it there from `data/street-cleaning.json`. The fetch URL `"data/street-cleaning.json"` resolves relative to `app/index.html`.

**Tests**
- GIVEN the app is in parked mode and the map is clicked, WHEN `getStreetName` resolves to `"Washington Street"`, THEN `showStreetPopup` is called with that road name and the matching cleaning entries.
- GIVEN the app is in parked mode and `getStreetName` returns `null` (e.g., tapped a park), THEN `showStreetPopup` is not called.
- GIVEN `street-cleaning.json` fails to load at startup, WHEN the map is clicked in parked mode, THEN `showStreetPopup` is still called with an empty entries array (no crash).
- GIVEN the app is in browsing mode and the map is clicked, THEN `getStreetName` is not called (position-setting branch runs instead).
