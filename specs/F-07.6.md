## F-07.6 Street Click Popup `[TODO]`

`showStreetPopup(lat: number, lng: number, streetName: string, entries: StreetCleaningEntry[]): void` — opens a Leaflet popup at the clicked coordinates. At most one street popup is open at any time; calling the function again closes the previous one. Dismissed automatically when the user clicks elsewhere (standard Leaflet behavior).

The block context comes from the `location` field of the matched entries (e.g., `"9th St. to 10th St."`). Format it as `"between 9th St and 10th St"` in the popup header. If multiple entries have different location values (different blocks of the same street), show each block as a separate group.

Street orientation determines which sides to show:
- N-S streets (Washington, Bloomfield, Garden, etc.) have East and West sides — show only E/W entries.
- E-W streets (1st through 14th St, Observer Hwy, Newark St, etc.) have North and South sides — show only N/S entries.
- The side is already present in `StreetCleaningEntry.side` — no orientation inference needed; just render whatever sides the entries carry.

Popup content layout:
- Bold street name + block context as header (e.g., `"Washington St between 9th St and 10th St"`)
- Subtitle: "Street Cleaning"
- Horizontal rule
- For each entry: side label (bold) + schedule text
- If `entries` is empty: "No cleaning schedule found" in muted text

Sign pin click handlers already call `L.DomEvent.stopPropagation`, so tapping a sign pin does not trigger `showStreetPopup`.

**Tests**
- GIVEN entries for a N-S street with East and West sides, THEN the popup contains the street name, "East" and "West" labels, and both schedule strings.
- GIVEN entries for an E-W street with North and South sides, THEN the popup contains the street name, "North" and "South" labels, and both schedule strings.
- GIVEN an entry whose location is "9th St. to 10th St.", THEN the popup header contains "between 9th St and 10th St".
- GIVEN `showStreetPopup` is called twice in succession, THEN only one popup exists on the map.
- GIVEN `entries` is an empty array, THEN the popup contains the street name and a "no schedule" message, and no error is thrown.
- GIVEN `initMap` has not been called, WHEN `showStreetPopup` is called, THEN it returns without throwing.
