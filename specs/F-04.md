## F-04 Shared Storage Module `[TODO]`

`shared/storage.ts` ŌĆö wraps `localStorage` behind an injectable interface so the module can be imported and tested in Node without a `window`. Implements save, load, and clear for one saved parking spot. **Must not call `localStorage` at module scope** ŌĆö only inside the returned object.

Tests live in `tests/unit/storage.test.ts`.

---

### F-04.1 Factory and StorageBackend Interface

Export `createSpotStorage(backend: StorageBackend): SpotStorage`.

`StorageBackend` interface: `{ getItem(key: string): string | null; setItem(key: string, value: string): void; removeItem(key: string): void }`.

`SpotStorage` interface: `{ save(spot: SavedSpot): void; load(): SavedSpot | null; clear(): void }`.

`SavedSpot` type: `{ lat: number; lng: number; side: "N" | "S" | "E" | "W"; savedAt: string; address: string | null }`.

**Tests**
- GIVEN `createSpotStorage` is called with a fake backend (a plain object backed by a `Map`), THEN it returns an object with `save`, `load`, and `clear` methods without throwing.
- GIVEN the fake backend, WHEN `save` is called, THEN subsequent `getItem` calls on the backend return a non-empty string.

---

### F-04.2 Save Spot

`storage.save(spot: SavedSpot): void` ŌĆö serializes `spot` to JSON and writes it to the backend under a fixed key (`"hoboken_parking_spot"`).

**Tests**
- GIVEN a valid `SavedSpot`, WHEN `save` is called, THEN `backend.getItem("hoboken_parking_spot")` returns a JSON string that `JSON.parse` deserializes to an object with the same `lat`, `lng`, `side`, and `savedAt`.
- GIVEN `save` is called twice with different spots, WHEN `load` is called, THEN the second spot is returned (not the first).

---

### F-04.3 Load Spot

`storage.load(): SavedSpot | null` ŌĆö reads and deserializes the stored spot. Returns `null` if the key is absent or if the stored value is not valid JSON.

**Tests**
- GIVEN nothing has been saved, WHEN `load` is called, THEN it returns `null`.
- GIVEN a valid spot was saved, WHEN `load` is called, THEN it returns the correct `SavedSpot` object.
- GIVEN the backend holds a malformed JSON string under the key, WHEN `load` is called, THEN it returns `null` without throwing.
- GIVEN the backend holds `null` under the key, WHEN `load` is called, THEN it returns `null`.

---

### F-04.4 Clear Spot

`storage.clear(): void` ŌĆö removes the spot from the backend.

**Tests**
- GIVEN a spot was saved, WHEN `clear` is called, THEN a subsequent `load` returns `null`.
- GIVEN nothing was saved, WHEN `clear` is called, THEN it does not throw.
