## F-16 Next Violation Window `[TODO]`

Adds `nextViolationWindow` and `ViolationWindow` to `shared/parking-logic.ts`. All existing exports in that file are preserved unchanged — this is an additive mutation only.

`SavedSpot` is not imported here (that would violate the pure-logic module constraint). The function takes an inline spot shape instead.

```typescript
export type ViolationWindow = {
  sign: Sign;
  minutesUntilActive: number;  // 0 if already active, positive if upcoming
}

export function nextViolationWindow(
  signs: Sign[],
  spot: { lat: number; lng: number },
  now: Date
): ViolationWindow | null
```

**Tests** (use `NOW_STABLE` from `tests/fixtures/signs.ts` for all `now` arguments)
- GIVEN no signs are within 150 m of the spot, WHEN `nextViolationWindow` is called with `now = NOW_STABLE`, THEN returns `null`.
- GIVEN one currently-active sign within 150 m (its window contains `NOW_STABLE`), WHEN called with `now = NOW_STABLE`, THEN returns `{ sign, minutesUntilActive: 0 }`.
- GIVEN one sign whose `start_iso` is exactly 90 minutes after `NOW_STABLE` and is within 150 m, WHEN called with `now = NOW_STABLE`, THEN returns `{ sign, minutesUntilActive: 90 }`.
- GIVEN two signs starting 90 and 45 minutes after `NOW_STABLE` respectively within 150 m, WHEN called with `now = NOW_STABLE`, THEN returns the sign with `minutesUntilActive: 45` (soonest).
- GIVEN one active sign and one upcoming sign within 150 m, WHEN called with `now = NOW_STABLE`, THEN returns the active sign with `minutesUntilActive: 0`.
- GIVEN all nearby signs expired before `NOW_STABLE`, WHEN called, THEN returns `null`.
- GIVEN a sign starting exactly `3 * 24 * 60` (4320) minutes after `NOW_STABLE` within 150 m, WHEN called with `now = NOW_STABLE`, THEN returns `{ sign, minutesUntilActive: 4320 }`.
- GIVEN an empty `signs` array, WHEN called, THEN returns `null` without throwing.

**Known data limitation:** The Hoboken API only returns signs that have already been posted (not future ones not yet in the system). A sign posted overnight will not appear until after the next 6 AM fetch. `nextViolationWindow` cannot warn about signs not yet in `latest.json`. This is documented behavior, not a bug.
