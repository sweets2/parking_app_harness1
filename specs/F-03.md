### F-03.2 haversineMeters

```typescript
export function haversineMeters(
  lat1: number, lng1: number,
  lat2: number, lng2: number
): number
```

Computes the great-circle distance in meters between two WGS84 coordinates. Must use the haversine formula (involves `Math.atan2`, `Math.sin`, `Math.cos`, and Earth radius 6,371,000 m). Euclidean approximation is not acceptable — at Hoboken's latitude (~40.75°) it produces ~12% error over 150 m distances.

**Tests**
- GIVEN two identical coordinates, THEN returns a value less than 0.001 (effectively zero).
- GIVEN `SIGN_PERMANENT_1` and `SIGN_11TH_ST` from fixtures (two signs known to be close together on the same street), THEN returns a value consistent with a short walking distance (under 50 m).
- GIVEN Hoboken's geographic center and `SIGN_BAD_COORD` from fixtures, THEN returns a value greater than 1000 (the bad-coord sign is far outside Hoboken).
- GIVEN `SIGN_13TH_ST_A` and `SIGN_13TH_ST_B` from fixtures (two signs at the same coordinates), THEN returns a value less than 2.

### F-03.3 isSignActive

```typescript
export function isSignActive(sign: Sign, now: Date): boolean
```

Returns `true` if `start_iso ≤ now ≤ end_iso` (both bounds inclusive). Uses `new Date(sign.start_iso).getTime()` and `new Date(sign.end_iso).getTime()` for comparison.

**Tests**
- GIVEN sign with `start_iso: "2026-05-26T08:00:00"` and `end_iso: "2026-05-29T16:00:00"`, WHEN `now = new Date("2026-05-28T12:00:00")`, THEN returns `true`.
- GIVEN the same sign, WHEN `now` equals exactly `new Date(sign.end_iso)`, THEN returns `true` (upper bound is inclusive).
- GIVEN the same sign, WHEN `now = new Date("2026-05-29T16:00:01")` (one second after end), THEN returns `false`.
- GIVEN sign with `end_iso: "2030-12-31T07:00:00"` (permanent sign), WHEN `now = new Date("2026-05-28T12:00:00")`, THEN returns `true`.
- GIVEN `SIGN_EXPIRED_1` from fixtures, WHEN `now = NOW_AFTER_EXPIRED`, THEN returns `false`.
- GIVEN `ACTIVE_SIGNS` from fixtures, WHEN `isSignActive` is called on each with `FETCH_TIME`, THEN every call returns `true` (0 mismatches with `active_at_fetch`).
- GIVEN `EXPIRED_SIGNS` from fixtures, WHEN `isSignActive` is called on each with `FETCH_TIME`, THEN every call returns `false` (0 mismatches with `active_at_fetch`).

### F-03.4 filterLoadTimeNoise

```typescript
export function filterLoadTimeNoise(signs: Sign[], now: Date): Sign[]
```

One-time startup filter. Removes two categories of signs:
1. Signs whose coordinates fall outside `HOBOKEN_BOUNDS` — regardless of their active status.
2. Signs where `active_at_fetch` is `false` AND `end_iso` is before `now` — definitively expired.

Signs with `active_at_fetch: false` but `end_iso` in the future are **kept** — they may be upcoming restrictions. `filterActive` handles them at runtime.

**Tests**
- GIVEN an array containing an expired sign (`active_at_fetch: false`, `end_iso` in the past relative to `now`), WHEN called, THEN the expired sign is not in the result.
- GIVEN `SIGN_BAD_COORD` from fixtures (outside `HOBOKEN_BOUNDS`), WHEN called with any `now`, THEN `SIGN_BAD_COORD` is not in the result.
- GIVEN `ALL_SIGNS` from fixtures, WHEN called with `FETCH_TIME`, THEN the result does not contain `SIGN_BAD_COORD`.
- GIVEN `ALL_SIGNS` from fixtures, WHEN called with `FETCH_TIME`, THEN the result contains `SIGN_PERMANENT_1`.
- GIVEN an array of only valid, active, in-bounds signs, THEN returns the full array unchanged.
- GIVEN an empty array, THEN returns an empty array.
- GIVEN a sign with `active_at_fetch: false` but `end_iso` one week in the future, THEN that sign IS in the result (upcoming, not expired).

### F-03.6 filterNearby

```typescript
export function filterNearby(
  signs: Sign[],
  lat: number,
  lng: number,
  radiusMeters: number,
  now: Date
): Sign[]
```

Returns signs that are both (a) within `radiusMeters` of `(lat, lng)` by haversine distance and (b) currently active per `isSignActive`. Both conditions must be true — a sign that is geographically close but expired must not appear, and an active sign beyond the radius must not appear.

The standard proximity radius for this app is 150 m.

**Tests**
- GIVEN a spot at the coordinates of `SIGN_PERMANENT_1` and radius 150 m, WHEN called with `ALL_SIGNS` and `NOW_STABLE`, THEN the result includes `SIGN_11TH_ST` (known to be nearby).
- GIVEN the same call, THEN `SIGN_BAD_COORD` is not in the result.
- GIVEN an empty array, THEN returns an empty array.
- GIVEN `radiusMeters = 0`, THEN returns an empty array (no sign is < 0 m away).
- GIVEN `SIGN_EXPIRED_1` is within the radius, WHEN called with `NOW_AFTER_EXPIRED`, THEN `SIGN_EXPIRED_1` is not in the result (expired despite being nearby).
- GIVEN a spot at the coordinates of `SIGN_13TH_ST_A` and radius 150 m, WHEN called with `NOW_STABLE`, THEN both `SIGN_13TH_ST_A` and `SIGN_13TH_ST_B` appear in the result (they share the same coordinates).

### F-03.7 formatCountdown

```typescript
export function formatCountdown(endIso: string, now: Date): string
```

Returns a human-readable time remaining string. Format: `"Xh Ym"` when hours ≥ 1, `"Ym"` when less than an hour, `"0m"` when zero or negative. Never returns a string containing a `-` character or a negative number.

**Tests**
- GIVEN `endIso = "2026-05-28T21:00:00"` and `now = new Date("2026-05-28T18:00:00")`, THEN returns `"3h 0m"`.
- GIVEN end is exactly 45 minutes after `now`, THEN returns `"45m"`.
- GIVEN end is exactly 90 minutes after `now`, THEN returns `"1h 30m"`.
- GIVEN end is 59 minutes and 59 seconds after `now`, THEN returns `"59m"` (seconds are truncated, not rounded).
- GIVEN `endIso` equals `now.toISOString()` (zero remaining), THEN returns `"0m"`.
- GIVEN `endIso` is before `now` (already expired), THEN returns `"0m"` and the result does not contain `-`.

### F-03.8 formatSignWindow

```typescript
export function formatSignWindow(sign: Sign, now: Date): string
```

Returns a string describing when the sign's restriction ends. If the end date is the same calendar day as `now` (compared using local date components), returns `"today at H:MM AM/PM"`. Otherwise returns `"Www Mmm D at H:MM AM/PM"` (e.g., `"Thu May 29 at 4:00 PM"`).

**Tests**
- GIVEN a sign with `end_iso: "2026-05-28T17:00:00"` and `now = new Date("2026-05-28T12:00:00")` (same day), THEN returns a string containing `"today"` and `"5:00 PM"`.
- GIVEN a sign with `end_iso: "2026-05-29T16:00:00"` and `now = new Date("2026-05-28T12:00:00")` (different day), THEN returns a string containing a weekday abbreviation (e.g., `"Fri"`) and `"4:00 PM"`.
- GIVEN a sign ending at midnight (`end_iso: "2026-05-29T00:00:00"`), THEN the time portion shows `"12:00 AM"`.

### F-03.10 nudgeCoords

```typescript
export function nudgeCoords(
  lat: number,
  lng: number,
  side: "N" | "S" | "E" | "W"
): { lat: number; lng: number }
```

Shifts a GPS coordinate by approximately 10 m in the given cardinal direction. The nudge magnitude is `0.00009` degrees — roughly 10 m at Hoboken's latitude (~40.75°). This is applied when saving a spot so the marker reflects which side of the street the car is on.

**Tests**
- GIVEN `side: "N"`, THEN `result.lat > input lat` and `result.lng === input lng`.
- GIVEN `side: "S"`, THEN `result.lat < input lat` and `result.lng === input lng`.
- GIVEN `side: "E"`, THEN `result.lng > input lng` and `result.lat === input lat`.
- GIVEN `side: "W"`, THEN `result.lng < input lng` and `result.lat === input lat`.
- GIVEN any side, THEN `Math.abs(result.lat - input lat)` or `Math.abs(result.lng - input lng)` is approximately `0.00009` (within 1e-7 tolerance).
