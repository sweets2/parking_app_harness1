## F-15 Manual Refresh Button `[TODO]`

A "Refresh signs" button visible only in parked mode. Shows data freshness. Tapping re-fetches and re-evaluates the saved spot.

---

### F-15.1 Button Visibility and Freshness Label

The button is visible in parked mode. Its label or subtitle shows when the data was last fetched: "Signs last updated today at 6:02 AM" or "Signs last updated Wed May 27 at 6:02 AM" for older data.

**Tests**
- GIVEN state is `"parked"`, WHEN the UI renders, THEN the refresh button is visible.
- GIVEN state is `"browsing"`, WHEN the UI renders, THEN the refresh button is not visible.
- GIVEN `fetched_at` is today at 06:02 UTC, WHEN the label renders, THEN it contains "today" and "6:02".
- GIVEN `fetched_at` is the previous day, WHEN the label renders, THEN it contains a weekday abbreviation and not "today".

---

### F-15.2 Tap to Refresh

Tapping the button: re-fetches `latest.json` with `cache: "no-cache"`, runs the same pipeline as startup (`filterLoadTimeNoise → filterActive → filterNearby`), and updates the banner and sign cards. The button shows a loading indicator while the fetch is in progress.

**Tests**
- GIVEN a new sign appeared since the last fetch, WHEN the refresh button is tapped, THEN the new sign appears in the nearby list after the fetch completes.
- GIVEN no new signs, WHEN the refresh button is tapped, THEN the banner and card list are unchanged.
- GIVEN the button is tapped and the network is unavailable, WHEN the fetch fails, THEN an error is shown briefly and the existing sign data remains displayed.
- GIVEN the button is tapped, THEN a loading state (spinner or disabled button) is visible during the in-flight request.
