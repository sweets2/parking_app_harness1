## F-01.9 Street Cleaning Data Scraper `[TODO]`

`fetcher/fetch-street-cleaning.ts` — a one-shot Node script (run manually, not daily) that scrapes the Hoboken street cleaning schedule and writes `data/street-cleaning.json`. The source is at `https://www.hobokennj.gov/resources/street-cleaning-schedule`. The data changes at most once a year; the file is committed to the repo and re-generated manually when the city updates the schedule.

**Page structure (important):** The page is built with Webflow CMS and does NOT use `<table>` elements. Each row is a `div[role="listitem"].w-dyn-item`. Inside each item there are two `div.table_wrapper` children — one has the class `w-condition-invisible` (a hidden mobile duplicate) and must be skipped. The visible wrapper contains four `div.table-content` children whose text content is Street, Side, Days & Hours, and Location respectively.

Correct selector chain: find all `div.w-dyn-item`, take the `div.table_wrapper` child that does NOT have `w-condition-invisible`, then read the four `div.table-content` children.

**Header row:** The column headers (Street / Side / Days & Hours / Location) appear as a live `div.w-dyn-item` row, not a `<thead>`. Filter it out by skipping any entry where `street === "Street"`.

Outputs a file conforming to `StreetCleaningData` (F-02.5). Unlike the daily sign fetcher, this script does not need a guard-before-write or count-change check — a human is running it intentionally.

**Tests**
- GIVEN the HTML page is fetched, WHEN the table is parsed, THEN every row produces an object with non-empty `street`, `side`, `schedule`, and `location` fields.
- GIVEN the parsed output, THEN `entries` contains more than 100 records.
- GIVEN the parsed output, THEN no entry has `street === "Street"` (header row is filtered).
- GIVEN a known row (e.g., Adams St. / West / Monday - 11 am to 12 noon / Newark St. to Sixteenth St.), WHEN parsed, THEN a matching entry exists in the output with the correct field values.
- GIVEN the fetch fails (network error), WHEN the script runs, THEN it exits non-zero without writing any file.
- GIVEN the script completes successfully, THEN `data/street-cleaning.json` is valid JSON conforming to `StreetCleaningData`.
