## F-05 GitHub Actions CI/CD Pipeline `[TODO]`

`.github/workflows/fetch_daily.yml` — the only production data refresh mechanism. Runs on a daily schedule. Enforces the pipeline order: **test → typecheck → fetch → commit if changed**. Netlify auto-deploys on push; no workflow step is needed for that.

Tests for this feature are verified by reading the workflow file, not by running it. An agent implementing this feature should write the YAML and then check each item below against the file.

---

### F-05.1 Scheduled Trigger

Cron runs at 11:00 UTC (`0 11 * * *`), which is 6:00 AM ET during EDT. Also has a `workflow_dispatch` trigger for manual runs.

**Verification checklist**
- The `on:` block contains a `schedule:` entry with cron `"0 11 * * *"` (or `"0 10 * * *"` for EST offset, with a comment explaining the choice).
- The `on:` block also contains `workflow_dispatch:`.

---

### F-05.2 Test Gate Before Fetch

`npm ci` and `npm test` run as the first job steps. If the test suite fails, the workflow fails and no fetch is attempted.

**Verification checklist**
- The `npm test` step appears before the `npm run fetch` step.
- There is no `continue-on-error: true` on the test step.

---

### F-05.3 Typecheck Gate

`npx tsc --noEmit` runs after tests and before the fetch. A type error stops the workflow.

**Verification checklist**
- A step running `tsc --noEmit` appears after `npm test` and before `npm run fetch`.

---

### F-05.4 Conditional Commit

After a successful fetch, run `git diff --quiet data/latest.json`. If the file changed, commit it with message `"data: update latest.json [skip ci]"` and push. If unchanged, skip the commit step.

**Verification checklist**
- The workflow uses `git diff --quiet` or equivalent to detect changes.
- The commit message contains `[skip ci]` (prevents the push from re-triggering the workflow).
- The step that commits only runs when the diff step detected a change.
- The workflow has `write` permissions on `contents` so it can push.
