## F-13 Data Freshness Check `[TODO]`

Lets the app and the user know when `latest.json` is older than expected — indicating that the GitHub Action likely failed.

---

### F-13.1 isDataStale Pure Function

`isDataStale(fetchedAt: string, now: Date): boolean` in `shared/parking-logic.ts`. Returns `true` if more than 25 hours have elapsed since `fetchedAt`. Returns `true` for unparseable input (fail-safe).

**Tests**
- GIVEN `fetchedAt` is 24 hours before `now`, WHEN `isDataStale` is called, THEN returns `false`.
- GIVEN `fetchedAt` is 26 hours before `now`, WHEN `isDataStale` is called, THEN returns `true`.
- GIVEN `fetchedAt` equals `now`, WHEN `isDataStale` is called, THEN returns `false`.
- GIVEN `fetchedAt` is 25 hours and 1 minute before `now`, WHEN `isDataStale` is called, THEN returns `true`.
- GIVEN `fetchedAt` is an empty string, WHEN `isDataStale` is called, THEN returns `true` (fail-safe — unknown freshness is treated as stale).
- GIVEN `fetchedAt` is not a valid date string, WHEN `isDataStale` is called, THEN returns `true` without throwing.

---

### F-13.2 Stale Data UI Warning

If `isDataStale` returns `true` at app load, display a banner: "Sign data may be outdated — last updated [N] hours ago." The banner does not prevent the app from being usable.

**Tests**
- GIVEN data is 30 hours old, WHEN the app loads, THEN a stale warning banner is visible.
- GIVEN data is 20 hours old, WHEN the app loads, THEN no stale warning banner is shown.
- GIVEN the stale warning is shown, THEN the rest of the app (map, signs, spot) still renders correctly.
