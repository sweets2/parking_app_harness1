## F-12 PWA Shell `[TODO]`

Makes the app installable to the home screen and functional offline. Requires `app/manifest.json` and `app/sw.ts` (compiled to `sw.js` by esbuild).

---

### F-12.1 Web App Manifest

`app/manifest.json` with: `name: "Hoboken Parking"`, `short_name: "HP Parking"`, `start_url: "/"`, `display: "standalone"`, `background_color`, `theme_color`, and at least two icon entries (192×192 and 512×512).

**Tests**
- GIVEN the manifest is fetched, THEN it is valid JSON.
- GIVEN the manifest, THEN `display` is `"standalone"`.
- GIVEN the manifest, THEN one icon entry has `sizes` containing `"192x192"`.
- GIVEN the manifest, THEN one icon entry has `sizes` containing `"512x512"`.

---

### F-12.2 Service Worker Registration

`app/main.ts` registers `sw.js` inside a `DOMContentLoaded` (or equivalent) handler, after `createApp` has been called. Registration errors are caught and logged but do not prevent the app from functioning. This must live in `main.ts`, not `app.ts` — `app.ts` must remain side-effect-free per CLAUDE.md.

**Tests** (file-content assertions — read `app/main.ts` as text)
- GIVEN `app/main.ts` source, THEN it contains `serviceWorker.register` and `sw.js`.
- GIVEN `app/main.ts` source, THEN the registration call is wrapped in a `try/catch` or `.catch(` error handler.

---

### F-12.3 App Shell Offline Cache

`app/sw.ts` caches the app shell on install: `index.html`, `app.js`, `style.css`, `manifest.json`, and the Leaflet CSS and JS from their CDN URL. Uses a cache-first strategy for the app shell. Cache name includes a version string so deploys flush the old cache.

**Tests** (file-content assertions — read `app/sw.ts` as text)
- GIVEN `app/sw.ts` source, THEN it contains all of: `'index.html'`, `'app.js'`, `'style.css'`, `'manifest.json'`.
- GIVEN `app/sw.ts` source, THEN it defines a `CACHE_NAME` constant whose value includes a version identifier (matches `/v\d/`).
- GIVEN `app/sw.ts` source, THEN it contains an `activate` handler that deletes caches not matching `CACHE_NAME`.

---

### F-12.4 latest.json Cache Strategy

`data/latest.json` uses a network-first strategy: try network, fall back to cached version if offline. On a successful network response, update the cache.

**Tests** (file-content assertions — read `app/sw.ts` as text)
- GIVEN `app/sw.ts` source, THEN it contains `'latest.json'` inside a fetch handler (network-first logic for that path).
- GIVEN `app/sw.ts` source, THEN it contains the exact string `"No data available — go online to load parking signs"` as the offline fallback message.
