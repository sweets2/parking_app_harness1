## F-12 PWA Shell `[TODO]`

Makes the app installable to the home screen and functional offline. Requires `app/manifest.json` and `app/sw.ts` (compiled to `sw.js` by esbuild).

---

### F-12.1 Web App Manifest

`app/manifest.json` with: `name: "Hoboken Parking"`, `short_name: "HP Parking"`, `start_url: "/"`, `display: "standalone"`, `background_color`, `theme_color`, and at least two icon entries (192×192 and 512×512).

**Tests**
- GIVEN the manifest is fetched, THEN it is valid JSON.
- GIVEN the manifest, THEN `display` is `"standalone"`.
- GIVEN the manifest, THEN one icon entry has `sizes` containing `"192x192"`.
- GIVEN the manifest, THEN one icon entry has `sizes` containing `"512x512"`.

---

### F-12.2 Service Worker Registration

`app/app.ts` registers `sw.js` after the app loads. Registration errors are caught and logged but do not prevent the app from entering `"browsing"` or `"parked"` state.

**Tests**
- GIVEN the browser supports service workers, WHEN the app loads, THEN `navigator.serviceWorker.register` is called with a path ending in `sw.js`.
- GIVEN service worker registration throws, WHEN the app loads, THEN the app still reaches a usable state.

---

### F-12.3 App Shell Offline Cache

`app/sw.ts` caches the app shell on install: `index.html`, `app.js`, `style.css`, `manifest.json`, and the Leaflet CSS and JS from their CDN URL. Uses a cache-first strategy for the app shell. Cache name includes a version string so deploys flush the old cache.

**Tests**
- GIVEN the app was loaded once online, WHEN the network is cut and the page is reloaded, THEN `index.html` and `app.js` are served from cache and the page loads.
- GIVEN the service worker is updated with a new cache name, WHEN it activates, THEN the old cache is deleted.

---

### F-12.4 latest.json Cache Strategy

`data/latest.json` uses a network-first strategy: try network, fall back to cached version if offline. On a successful network response, update the cache.

**Tests**
- GIVEN the app is online, WHEN `latest.json` is fetched, THEN the response is stored in cache.
- GIVEN the app is offline and a cached `latest.json` exists, WHEN the app loads, THEN the cached version is returned and the stale-data warning (F-13.2) is shown.
- GIVEN the app is offline and no cache exists for `latest.json`, WHEN the app loads, THEN an error message "No data available — go online to load parking signs" is shown.
