## F-02.5 Street Cleaning Types `[TODO]`

Add `StreetCleaningEntry` and `StreetCleaningData` to `shared/types.ts`.

```typescript
export interface StreetCleaningEntry {
  street:   string;   // "Washington Street"
  side:     string;   // "East" | "West" | "North" | "South" | "Both"
  schedule: string;   // "Monday through Friday - 8 am to 9 am"
  location: string;   // "Observer Hwy. to Seventh St."
}

export interface StreetCleaningData {
  fetched_at: string;                // ISO 8601 UTC — when the scrape ran
  entries:    StreetCleaningEntry[];
}
```

`side` is typed as `string` rather than a union because the city's HTML table may use varied casing or phrasing; normalization is the scraper's concern, not the type's. `schedule` preserves the raw text exactly as scraped — it is displayed to the user verbatim, never parsed for time logic.

**Tests** (compiler-level)
- GIVEN a `StreetCleaningEntry`, THEN all four fields are required (no optional fields).
- GIVEN a `StreetCleaningData` object, THEN `entries` is typed as `StreetCleaningEntry[]`, not `any[]`.
