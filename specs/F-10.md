## F-10 Flow 1 — Just Parked (End-to-End) `[TODO]`

The complete path a user takes immediately after parking. The user taps the map to set their position — there is no GPS. Requires F-06, F-07, F-09 to be functional.

---

### F-10.1 Open App Cold (No Saved Spot)

**Tests**
- GIVEN no saved spot, WHEN the app opens, THEN the map is visible within 3 seconds on a normal network.
- GIVEN no saved spot, THEN the app enters `"browsing"` state with no position marker on the map.

---

### F-10.2 Tap to Set Position

The user taps anywhere on the map to indicate where they are. `registerMapClickHandler` (from F-07) is wired in `main.ts`. In browsing mode, a tap calls `renderPositionMarker` and stores the tapped coordinates as the current position.

**Tests**
- GIVEN the app is in browsing mode, WHEN the map is tapped at (40.744, -74.032), THEN a position marker appears at those coordinates.
- GIVEN the map is tapped a second time at different coordinates, THEN the position marker moves to the new location (only one marker at a time).
- GIVEN no tap has occurred, WHEN SAVE MY SPOT is tapped, THEN nothing happens (button is disabled or shows an error — no position to save).

---

### F-10.3 Active Sign Pins Visible on Map

**Tests**
- GIVEN `latest.json` has 67 active signs at the time of loading, WHEN the app enters browsing mode, THEN 67 sign pins are visible on the map.
- GIVEN a CONSTRUCTION and a DELIVERY sign are both active, THEN the CONSTRUCTION pin is red and the DELIVERY pin is blue.
- GIVEN a sign pin is tapped, THEN a popup shows the reason, address, and active window string.

---

### F-10.4 Save My Spot

**Tests**
- GIVEN the user has tapped a position, WHEN SAVE MY SPOT is tapped, THEN the street-side picker appears.
- GIVEN the user selects "N", WHEN the spot is saved, THEN the saved spot's `lat` is the tapped lat plus approximately 0.00009 degrees, and `lng` is unchanged.
- GIVEN the user selects "E", WHEN the spot is saved, THEN the saved spot's `lng` is the tapped lng plus approximately 0.00009 degrees, and `lat` is unchanged.
- GIVEN the spot is saved, THEN the app transitions to `"parked"` state.
- GIVEN the spot is saved, THEN a spot marker appears on the map and sign pins update to reflect nearby-only view.
- GIVEN the spot is saved, THEN a confirmation toast is shown briefly.
- GIVEN the app is closed and reopened after saving, THEN it opens in `"parked"` mode with the same spot (persisted in localStorage).
