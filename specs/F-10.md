### F-10.2 Tap to Set Position

The user taps anywhere on the map to indicate where they are. `registerMapClickHandler` (from F-07) is wired in `main.ts`. In browsing mode, a tap calls `renderPositionMarker` and stores the tapped coordinates as the current position.

**Tests**
- GIVEN the app is in browsing mode, WHEN the map click handler fires with (40.744, -74.032), THEN `renderPositionMarker` is called with those exact coordinates.
- GIVEN the map click handler fires twice at different coordinates, THEN `renderPositionMarker` is called twice — only one marker tracks the latest position.
- GIVEN no tap has occurred (userLat/userLng are null), WHEN SAVE MY SPOT is tapped, THEN `showStreetSidePicker` is not called (save is suppressed).

### F-10.3 Active Sign Pins Visible on Map

`app/map.ts` must export `signColor(reason: string): string` — a pure function returning a hex color string for a given sign reason. The existing `REASON_COLOR` map in `app/map.ts` is the source of truth; `signColor` must wrap it. `renderSignPins` uses this to color each marker.

The existing `REASON_COLOR` values in `app/map.ts` are:
- `"CONSTRUCTION"` → `"#e53e3e"`
- `"DELIVERY"` → `"#3182ce"`
- Unknown reason → `"#718096"` (grey fallback)

**Tests**
- GIVEN `signColor("CONSTRUCTION")`, THEN it returns `"#e53e3e"`.
- GIVEN `signColor("DELIVERY")`, THEN it returns `"#3182ce"`.
- GIVEN `signColor("UNKNOWN_REASON")`, THEN it returns `"#718096"` (grey fallback).
- GIVEN the app is in browsing mode with 3 active signs, WHEN `renderState` fires, THEN `renderSignPins` is called with an array of length 3 (mock-based: `vi.mock("../../app/map")`).
- GIVEN `app/map.ts` source, THEN `renderSignPins` contains popup-building code that references `sign.address` and `sign.reason`.
