### F-00.6 App HTML Shell

`app/index.html` вЂ” the single HTML page. Contains all structural elements the UI and map modules expect to find in the DOM. No logic. No inline scripts. Links to `style.css` and loads `app.js` as a module.

Required elements and attributes:
- `<meta name="viewport" content="width=device-width, initial-scale=1">` вЂ” prevents zoom on mobile
- `<link rel="manifest" href="manifest.json">` вЂ” PWA install support
- `<link rel="stylesheet" href="style.css">`
- `<div id="map"></div>` вЂ” Leaflet attaches here
- `<div id="banner"></div>` вЂ” clear/warning/stale banners render here
- `<div id="sign-list"></div>` вЂ” sign cards render here
- `<button id="save-btn">Save My Spot</button>`
- `<button id="here-btn">I'm Here Now</button>`
- `<button id="clear-btn">Clear My Spot</button>`
- `<script type="module" src="app.js"></script>` вЂ” at end of body

**Tests** (verified by reading the file вЂ” no Vitest needed)
- GIVEN the file, THEN it contains `<div id="map">`.
- GIVEN the file, THEN it contains `<div id="banner">`.
- GIVEN the file, THEN it contains `<div id="sign-list">`.
- GIVEN the file, THEN it contains `id="save-btn"`, `id="here-btn"`, and `id="clear-btn"`.
- GIVEN the file, THEN it links to `style.css` via a `<link>` tag.
- GIVEN the file, THEN it loads `app.js` with `type="module"`.
- GIVEN the file, THEN it contains a viewport `<meta>` tag.
- GIVEN the file, THEN it links to `manifest.json`.

### F-00.7 App Stylesheet

`app/style.css` вЂ” mobile-first layout. No frameworks. The map must occupy the majority of the screen height. Buttons must be at the bottom, thumb-reachable. Sign cards must be scrollable.

Required layout rules:
- `html, body` вЂ” `margin: 0; padding: 0; height: 100%; font-family: sans-serif`
- `#map` вЂ” fixed height using `calc(100vh - [bottom panel height])`, e.g. `calc(100vh - 180px)`
- `#sign-list` вЂ” `overflow-y: auto` with a max-height so it scrolls within its panel
- Button container вЂ” positioned at the bottom, full width, with sufficient padding for thumbs
- `#here-btn` and `#clear-btn` вЂ” hidden by default (`display: none`); shown by `app.ts` when in parked mode
- `#banner` вЂ” hidden by default (`display: none`); shown by `ui.ts` when a banner state is active

**Tests** (verified by reading the file вЂ” no Vitest needed)
- GIVEN the file, THEN `#map` has an explicit height rule.
- GIVEN the file, THEN `#sign-list` has `overflow-y: auto` or `overflow-y: scroll`.
- GIVEN the file, THEN `#here-btn` and `#clear-btn` have `display: none` as their default rule.
- GIVEN the file, THEN `#banner` has `display: none` as its default rule.
