## F-33 Intersection-Based Segment Geocoding `[DONE]`

Fixes a bug where clicking 7th Street west of Adams highlights the wrong street-cleaning block.

### Root Cause

`detectMatchingSegment` in `shared/parking-logic.ts` chooses the latitude or longitude axis by
comparing `deltaLat` vs `deltaLng` between the two cross-street geocoded coordinates. The geocoded
coordinates came from standalone avenue centroids (e.g. Hudson Street centroid at lat 40.7536),
not from the actual intersection with the clicked street (lat ≈ 40.7462 where Hudson crosses 7th).

For the "Willow Ave. to Hudson St." segment on 7th Street:
- Willow Ave centroid: lat 40.7371  
- Hudson St centroid: lat 40.7536  
- deltaLat = 0.0165 > deltaLng = 0.0100 → wrong: latitude axis chosen  
- 7th Street runs at lat ≈ 40.7462, squarely between those two centroids  
- Result: every click on 7th Street matches "Willow to Hudson" regardless of block

### Fix

Synthesize intersection coordinates from standalone centroids using road orientation:

- **East-west main roads** (numbered streets, Observer Hwy, Newark St, Sinatra Dr):  
  `intersection = { lat: mainCoord.lat, lng: crossCoord.lng }`  
  Gives deltaLat ≈ 0 → forces longitude axis → correct block detection.

- **North-south main roads** (named avenues, named streets):  
  `intersection = { lat: crossCoord.lat, lng: mainCoord.lng }`  
  Gives deltaLng ≈ 0 → forces latitude axis → correct block detection.

No change to `detectMatchingSegment` itself — only the coordinates fed into it change.

### `fetcher/geocode-cross-streets.ts` — modifications

- In `runGeocode()`, add `normalizeStreet(entry.street)` to the standalone `names` Set so all
  main streets are geocoded in Phase 1 (not only cross-streets).
- Add `isEastWestRoad(name: string): boolean` helper that returns true for numbered streets
  and known E-W roads.
- Add Phase 2 loop over `(main, cross)` intersection pairs:
  - Key: `"${main}|${cross}"`
  - If cross is a STATIC_COORDS entry: reuse static value
  - Otherwise: synthesize from Phase 1 standalone coords using `isEastWestRoad(main)`
- Collect intersection pairs by iterating entries and adding both `(main, from)` and `(main, to)`
  from each `entry.location`.

### `app/geo.ts` — modifications

Change `geocodeCrossStreet(streetName: string)` to
`geocodeCrossStreet(streetName: string, mainStreet?: string)`:

- When `mainStreet` is provided:
  - Cache key: `"${mainStreet}|${streetName}"`
  - Query: `"${streetName} & ${mainStreet}, Hoboken, NJ"`
  - If intersection key not in cache but standalone key IS in cache: use standalone as fallback
    (preserves existing behavior for streets not pre-seeded with intersection data)
- When `mainStreet` is omitted: unchanged (existing callers unaffected)

### `app/main.ts` — modifications

Add `roadName: string` as third parameter to `buildDetectSegmentCallback`. Pass
`normalizeStreet(roadName)` as the second argument to both `geocodeCrossStreet` calls inside
the returned async function. Update both call sites (parked state on load, map click handler).
