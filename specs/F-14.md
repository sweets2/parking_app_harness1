## F-14 Flow 3 — Automatic Re-Fetch on Open `[TODO]`

When a saved spot exists and the app opens, re-fetch `latest.json` with `cache: "no-cache"` before rendering the parked view. This is wired entirely in `app/main.ts` (`initBrowserApp`). **`app/app.ts` is not modified** — doing so would violate CLAUDE.md (state machine only, no side effects).

### Module interface — modified symbols

This feature modifies `initBrowserApp` in `app/main.ts`. The function signature does not change:

```typescript
export async function initBrowserApp(): Promise<void>
```

The behavioral change: after a successful initial fetch, if `storage.load()` returns a non-null saved spot, perform a second fetch of `data/latest.json` with `{ cache: "no-cache" }`. Use the freshly fetched signs if the second fetch succeeds; fall back to the first fetch's signs if it fails. All existing initialization behavior is preserved.

---

### F-14.1 Cache-Bypassing Re-Fetch

**Tests**
- GIVEN a saved spot exists in storage, WHEN `initBrowserApp` runs, THEN `fetch` is called at least twice — the second call uses `{ cache: "no-cache" }` (verified via `vi.spyOn(globalThis, 'fetch')` capturing call arguments).
- GIVEN a saved spot exists and the no-cache fetch fails with a network error, WHEN `initBrowserApp` runs, THEN `createApp` is still called, the mock app reaches `"parked"` state, and no unhandled error is thrown.
- GIVEN a saved spot exists and the no-cache fetch returns a JSON body with 5 signs, WHEN `createApp` is called, THEN the `initialData.signs` argument has length 5 (the fresh fetch result, not any prior cached value).

---

### F-14.2 Warning-First Render

The existing `renderState` callback already transitions directly to the warning banner when `nearbySigns.length > 0`. No new rendering logic is needed — the only requirement is that the fresh signs are passed to `createApp` before `renderState` is first called.

**Tests**
- GIVEN a saved spot exists and the no-cache fetch returns a sign that has `start_iso` before `NOW_STABLE` and `end_iso` after `NOW_STABLE` and coordinates within 150 m of the saved spot, WHEN `initBrowserApp` runs with `now = NOW_STABLE`, THEN `renderWarningBanner` is called (not `renderClearBanner`) — verified via `vi.mock("../../app/ui")`.
- GIVEN a saved spot exists and the no-cache fetch returns no signs within 150 m, WHEN `initBrowserApp` runs, THEN `renderClearBanner` is called.
