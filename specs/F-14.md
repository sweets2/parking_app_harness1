## F-14 Flow 3 — Automatic Re-Fetch on Open `[DONE]`

When a saved spot exists and the app opens, re-fetch `latest.json` before rendering the parked view. If a new active sign appeared near the saved spot overnight, the first thing the user sees is the warning — not a clean map.

---

### F-14.1 Cache-Bypassing Re-Fetch

When transitioning from `"loading"` to `"parked"` state, fetch `latest.json` with `cache: "no-cache"`. Update the sign list from the fresh response before calling any render function.

**Tests**
- GIVEN a saved spot exists, WHEN the app opens, THEN the fetch request includes `cache: "no-cache"` (not the default browser cache behavior).
- GIVEN the re-fetch succeeds, WHEN the parked view renders, THEN it uses the freshly fetched signs, not a stale in-memory copy from a previous session.
- GIVEN the re-fetch fails (offline), WHEN the app loads, THEN it falls back to the cached `latest.json`, shows the stale-data warning, and still renders the parked view.

---

### F-14.2 Warning-First Render

If the freshly fetched data contains active signs within 150 m of the saved spot, the loading screen transitions directly to the warning banner without flashing the map first.

**Tests**
- GIVEN a new CONSTRUCTION sign appeared since the last session within 100 m of the saved spot, WHEN the app finishes loading, THEN the warning banner is visible before any map tiles are rendered.
- GIVEN no new signs near the saved spot, WHEN the app finishes loading, THEN the map renders first, then the clear banner appears.
