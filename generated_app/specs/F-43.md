# F-43 — Fix: Move Tow Sign Pin Markers Further from Road onto Property Side

## What this is

F-42 offset pin markers by `LATERAL_OFFSET_M = 4.0` m — the same distance used for
the tow-zone polylines. This places the pins at the curb, visually on top of the road
segment lines. This feature increases the pin offset so markers sit on the **property
side of the sidewalk**, clearly separated from the road polylines.

The polyline offset (`LATERAL_OFFSET_M = 4.0`) stays unchanged. Only the pin marker
offset increases.

---

## Module contract

### `app/map.ts`

**Add one new exported constant** near the existing `LATERAL_OFFSET_M` constant:

```typescript
export const PIN_LATERAL_OFFSET_M = 14.0;
```

14 m from the road centerline clears the half-street (~4.5 m) plus the sidewalk
(~3 m) in a typical Hoboken street and places the marker visibly on the adjacent
building face.

**Update `getSnappedPinPosition`** — replace both uses of `LATERAL_OFFSET_M` in the
lateral-offset block with `PIN_LATERAL_OFFSET_M`:

```typescript
projPt = [
  projPt[0] + dir * PIN_LATERAL_OFFSET_M * perpY / 111320,
  projPt[1] + dir * PIN_LATERAL_OFFSET_M * perpX / (111320 * cosLat2),
];
```

No other changes to `getSnappedPinPosition`. No changes to `renderTowSegments` or
`renderSignPins` callers. No changes to `app/main.ts`.

---

## Known gotchas

**Only change the pin offset, not the polyline offset.** `renderTowSegments` calls
`offsetPolylinePoints(..., LATERAL_OFFSET_M, ...)` — leave those calls using
`LATERAL_OFFSET_M`. Only `getSnappedPinPosition` should use `PIN_LATERAL_OFFSET_M`.

**Export the constant.** Mark it `export const PIN_LATERAL_OFFSET_M` so tests can
import it to compute expected displacements without hardcoding `14.0`.

**`now` in test snippets means `NOW_STABLE`.** Use `NOW_STABLE` from
`tests/fixtures/signs.ts` wherever the spec shows `now` — never use `new Date()`.

**`MockLayer` in test snippets means `MockMarker`.** The existing test file defines
`MockMarker`, not `MockLayer`. Use `MockMarker` as the type annotation in filter callbacks.

## Tests

Tests are in `tests/unit/map.test.ts` under
`describe("F-43 PIN_LATERAL_OFFSET_M pin distance")`.

Setup:

```typescript
// N-S road segment: A = [40.740, -74.030], B = [40.741, -74.030]
// Right-perp: perpX = 1 (east), perpY = 0
// Odd address → dir=1 → pin shifts east (lng increases)
initRoadGeometry({ "BLOOMFIELD ST": [[[40.740, -74.030], [40.741, -74.030]]] });
initStreetParity({ "BLOOMFIELD ST": 1 });
```

**Given** a sign `{ address: "101 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }` (odd),
**When** `renderSignPins([sign], NOW_STABLE)` is called,
**Then** the pin's lng offset from centerline should be approximately
`PIN_LATERAL_OFFSET_M / (111320 * cos(40.7405 * PI / 180))`.

**Given** the same setup,
**When** `renderSignPins([sign], NOW_STABLE)` is called,
**Then** the pin's offset from centerline should be strictly greater than
`LATERAL_OFFSET_M / (111320 * cos(lat))` (polyline offset).

**Given** a sign `{ address: "100 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }` (even),
**When** `renderSignPins([sign], NOW_STABLE)` is called,
**Then** the pin shifts west (lng decreases) by `PIN_LATERAL_OFFSET_M`.
