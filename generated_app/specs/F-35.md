## F-35 Upcoming Tow Zone Signs (Orange Layer)

A second map layer showing tow zones that are scheduled but not yet active,
displayed in orange (`#f97316`). Mirrors the existing red active-sign layer
exactly: same legend toggle layout, same polyline casing style, same popup
content. Data comes from a new `data/future.json` file written by the fetcher.

Upcoming orange markers render in a custom Leaflet pane (`upcomingPane`,
zIndex 550) so they are visually isolated from `.leaflet-marker-pane`. This
means the active-tow CSS toggle (`#map.tow-signs-hidden .leaflet-marker-pane`)
does **not** affect upcoming markers, and both toggles work fully independently.

Files modified:

- `fetcher/fetch.ts` — `runFutureFetcherWithFs`, chained from `runFetcherWithFs`
- `app/map.ts` — `UPCOMING_SIGN_ICON`, new state vars, `renderUpcomingSignPins`, `renderUpcomingTowSegments`, `setUpcomingSignsVisible`
- `app/main.ts` — `upcomingSignsData`, future.json fetch, render calls, legend wiring, silentRefresh update
- `app/index.html` — add `#upcoming-legend` between `#tow-legend` and `#violation-legend`
- `app/style.css` — upcoming legend styles; shift `#violation-legend` top from 168px → 210px
- `package.json` — build script copies `data/future.json` to `app/data/future.json`
- `tests/unit/map.test.ts` — 10 test cases
- `tests/unit/fetcher.test.ts` — 5 test cases

### `fetcher/fetch.ts` — modifications

Add after existing `API_URL`:

```typescript
const FUTURE_API_URL = "https://api-hpuvp.hobokennj.gov/api/v1/parking/future";
```

**New exported function `runFutureFetcherWithFs`:**

```typescript
export async function runFutureFetcherWithFs(
  fetchTime: Date,
  fsBackend: FsBackend
): Promise<void>
```

- Fetches from `FUTURE_API_URL`; calls `process.exit(1)` on HTTP error (same pattern as `runFetcherWithFs`)
- Reuses `validateResponseShape`, `validateSign`, `transformSign`
- Builds `fetchLocalIso` the same way `computeActiveAtFetch` does (UTC-based "YYYY-MM-DDTHH:MM:SS" string)
- Keeps only signs where `sign.start_iso > fetchLocalIso` (upcoming only; excludes currently active)
- Writes `data/future.json` with `{ fetched_at, count, signs }` — no archive file
- **Zero upcoming signs is valid** — writes empty array, does NOT call `process.exit`

Chain at end of `runFetcherWithFs`, after writing `latest.json`:

```typescript
await runFutureFetcherWithFs(fetchTime, fsBackend);
```

### `app/map.ts` — modifications

**New constant** `UPCOMING_SIGN_ICON` — a standalone orange no-parking SVG icon.
The active tow sign layer currently uses `signEmoji()` (emoji string); upcoming signs
use a distinct SVG icon so they are visually unambiguous at a glance:

```typescript
const UPCOMING_SIGN_ICON =
  `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100" width="20" height="20">` +
  `<circle cx="50" cy="50" r="43" fill="none" stroke="#f97316" stroke-width="13"/>` +
  `<text x="50" y="70" text-anchor="middle" font-family="Arial Black,Impact,sans-serif" font-size="62" font-weight="900" fill="#1a1a1a">P</text>` +
  `<rect x="-8" y="44" width="116" height="11" rx="5" fill="#f97316" transform="rotate(-35 50 50)"/>` +
  `</svg>`;
```

**New module state** (add after `_violationHighlightsVisible`):

```typescript
let _upcomingSignLayers: L.Marker[] = [];
let _upcomingSegmentLayers: L.Polyline[] = [];
let _upcomingSignsVisible: boolean = true;
```

Reset all three in `initMap` alongside existing resets.

**`initMap()` — custom pane setup** (add after `_map = map;` and existing resets):

```typescript
_upcomingSignLayers = [];
_upcomingSegmentLayers = [];
_upcomingSignsVisible = true;
map.createPane('upcomingPane');
const upcomingPaneEl = map.getPane('upcomingPane');
if (upcomingPaneEl !== undefined) upcomingPaneEl.style.zIndex = '550';
```

The custom pane has zIndex 550 — above `overlayPane` (400) but below `markerPane` (600). This means upcoming orange pins render above street geometry but below active tow pins. Critically, the active-tow CSS rule `#map.tow-signs-hidden .leaflet-marker-pane { display: none }` does **not** affect `upcomingPane`, so the two toggles are fully independent.

**`renderUpcomingSignPins(signs: Sign[], now: Date): void`**

- Clear `_upcomingSignLayers`: remove each layer from map, reset to `[]`
- Loop signs; create `L.marker([sign.lat, sign.lng], { icon: L.divIcon({ html: UPCOMING_SIGN_ICON, ... }), pane: 'upcomingPane' })` — the `pane: 'upcomingPane'` option places the marker in the custom pane
- Bind popup via `buildSignPopup(sign, now)` (unchanged — popup already handles upcoming-soon badge)
- If `_upcomingSignsVisible` is true, call `marker.addTo(_map)`
- Push each marker to `_upcomingSignLayers`

**`renderUpcomingTowSegments(signs: Sign[]): void`**

- Same geometry logic as `renderTowSegments` (road-geometry lookup → heuristic fallback → lateral offset)
- Polyline styles: outer `{ color: '#fff', weight: 7, opacity: 0.65 }`, inner `{ color: '#f97316', weight: 3, opacity: 0.85 }`
- Clear `_upcomingSegmentLayers` before rendering (remove + reset `[]`)
- Push to `_upcomingSegmentLayers`; add to map only if `_upcomingSignsVisible`

**`setUpcomingSignsVisible(visible: boolean): void`**

- Set `_upcomingSignsVisible = visible`
- If `_map === null` return
- Call `_map.closePopup()` when hiding
- Loop both `_upcomingSignLayers` and `_upcomingSegmentLayers` imperatively (`layer.addTo(_map)` / `layer.remove()`)

### `app/main.ts` — modifications

**Module-level state**: add `let upcomingSignsData: Sign[] = [];` alongside `cleaningEntries`.

**Imports**: add `renderUpcomingSignPins`, `renderUpcomingTowSegments`, `setUpcomingSignsVisible` from `./map`.

**In `initBrowserApp`**, after the `latest.json` fetch block, add a silent-catch fetch of `data/future.json`:

```typescript
try {
  const futureRes = await fetch("data/future.json");
  const futureJson = await futureRes.json() as { fetched_at: string; signs: Sign[] };
  upcomingSignsData = filterLoadTimeNoise(futureJson.signs, new Date(futureJson.fetched_at))
    .filter((s) => !isSignActive(s, now));
} catch {
  // file missing or network error — layer stays empty
}
```

**In `renderState`** (both browsing and parked branches), after existing `renderTowSegments` calls:

```typescript
renderUpcomingSignPins(upcomingSignsData, now);
renderUpcomingTowSegments(upcomingSignsData);
```

**In `silentRefresh`**, re-fetch `data/future.json` with `cache: "no-cache"`, apply the same `filterLoadTimeNoise + !isSignActive` filter, update `upcomingSignsData`, then call `renderUpcomingSignPins` and `renderUpcomingTowSegments`.

**Legend wiring** (after the existing violation-toggle block):

```typescript
const upcomingToggle = document.getElementById("upcoming-toggle");
upcomingToggle?.addEventListener("click", () => {
  const isOn = upcomingToggle.getAttribute("aria-pressed") === "true";
  const next = !isOn;
  setUpcomingSignsVisible(next);
  upcomingToggle.setAttribute("aria-pressed", String(next));
  document.getElementById("upcoming-legend")?.classList.toggle("upcoming-off", !next);
  const status = document.getElementById("upcoming-legend")?.querySelector(".upcoming-status");
  if (status !== null && status !== undefined) {
    status.textContent = next ? "Enabled" : "Hidden";
  }
});
```
