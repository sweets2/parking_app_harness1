# F-42 тАФ Fix: Offset Tow Sign Pin Markers to Curb

## What this is

Companion fix to F-41. F-41 offset the polyline segments to the curb but left the
pin markers (the `!` circle SVG icons) snapped to the road centerline. This feature
applies the same parity-based lateral offset inside `getSnappedPinPosition` so that
pin markers also appear at the curb side of the road.

Additionally, `app/main.ts` must await `data/street-parity.json` before the first
`renderSignPins` call тАФ otherwise `_streetParity` is still `{}` during initial render
and no pins are offset. The current `main.ts` already has this fix (parity fetch moved
into `Promise.all` with road-geometry); the Creator must preserve that structure.

## Module contract

### `getSnappedPinPosition` (private, in `app/map.ts`)

Current signature (unchanged):
```typescript
function getSnappedPinPosition(sign: Sign): [number, number]
```

**New behavior:** after projecting `sign.lat/lng` onto the nearest road segment
centerline, apply a lateral offset using the same parity + perpendicular logic as
`renderTowSegments`.

**Algorithm change тАФ save best segment endpoints:**

The current implementation saves only `projPt` and `bestDist`. To compute the
perpendicular after finding the best segment, it must also save the segment endpoints
so it can derive the road direction. Add `bestA` and `bestB` variables alongside
`projPt`:

```typescript
let bestA: [number, number] = [sign.lat, sign.lng];
let bestB: [number, number] = [sign.lat, sign.lng];
```

Update inside the inner loop when `d < bestDist`:
```typescript
bestDist = d;
projPt = [A[0] + t * (B[0] - A[0]), A[1] + t * (B[1] - A[1])];
bestA = A;
bestB = B;
```

**After the loop, apply lateral offset:**

```typescript
// Apply parity-based lateral offset to move pin from centerline to curb.
const numMatch = sign.address.match(/^(\d+)/);
if (numMatch !== null) {
  const streetKey = normalizeToGeometryKey(
    sign.address.replace(/^\d[\d-]*\s+/, "").trim()
  );
  const oddDir = _streetParity[streetKey];
  if (oddDir !== undefined) {
    const isOdd = parseInt(numMatch[1], 10) % 2 === 1;
    const dir: 1 | -1 = isOdd ? oddDir : (oddDir === 1 ? -1 : 1);
    const cosLat2 = Math.cos(projPt[0] * Math.PI / 180);
    const dYseg = bestB[0] - bestA[0];
    const dXseg = (bestB[1] - bestA[1]) * cosLat2;
    const lenSeg = Math.sqrt(dYseg * dYseg + dXseg * dXseg);
    if (lenSeg > 0) {
      // Right-perpendicular тАФ same convention as offsetPolylinePoints
      const perpX = dYseg / lenSeg;   // east component
      const perpY = -dXseg / lenSeg;  // north component
      projPt = [
        projPt[0] + dir * LATERAL_OFFSET_M * perpY / 111320,
        projPt[1] + dir * LATERAL_OFFSET_M * perpX / (111320 * cosLat2),
      ];
    }
  }
}
return projPt;
```

**No change to callers.** `renderSignPins` and `renderUpcomingSignPins` call
`getSnappedPinPosition(sign)` тАФ no call-site changes needed.

### `app/main.ts` (modified тАФ timing fix must be preserved)

The `Promise.all` block that awaits both `road-geometry.json` and `street-parity.json`
before `createApp()` runs must be preserved exactly:

```typescript
await Promise.all([
  fetch("data/road-geometry.json")
    .then((r) => r.json())
    .then((g: RoadGeometry) => { initRoadGeometry(g); })
    .catch(() => { /* non-fatal */ }),
  fetch("data/street-parity.json")
    .then((r) => r.json())
    .then((data: unknown) => { initStreetParity(data as Record<string, 1 | -1>); })
    .catch(() => { /* non-fatal */ }),
]);
```

There must be NO separate fire-and-forget `fetch("data/street-parity.json")` call
after `createApp()` тАФ that duplicate was removed when the timing fix was applied.

## Known gotchas

**Save bestA and bestB in the loop.** The current loop body only updates `projPt` and
`bestDist`. You must also update `bestA = A` and `bestB = B` in the same `if (d < bestDist)`
block тАФ otherwise the perpendicular is computed from the default (0,0) fallback values.

**Use `normalizeToGeometryKey` for the street lookup.** `getSnappedPinPosition` already
extracts the street name with `sign.address.replace(/^\d[\d-]*\s+/, "").trim()`. Pass
that result through `normalizeToGeometryKey()` before looking it up in `_streetParity` тАФ
the parity map keys are normalized (e.g. `"8TH ST"` not `"EIGHTH ST"`).

**Right-perpendicular convention must match `offsetPolylinePoints`.** Use
`perpX = dY/len, perpY = -dX/len` (90┬░ CW). Do NOT use the left-perpendicular
(`-dY/len, dX/len`) from the pre-F-41 version.

**`normalizeToGeometryKey` is not exported.** It is a module-private function. The
call inside `getSnappedPinPosition` is within the same file, so this is fine.

## Tests

Add a new `describe("F-42 getSnappedPinPosition parity offset")` block to
`tests/unit/map.test.ts`.

**How to access markers in tests:**
- Regular tow sign pins are added with `pane: 'towSignPane'`. Find them via:
  `mockMapInstance._layers.filter(l => l._options["pane"] === "towSignPane")`
- Upcoming sign pins use `pane: 'upcomingPane'`.
- Marker coordinates are on `._lat` and `._lng` (not `getLatLng()`).

Setup used across all F-42 tests:
```typescript
// N-S road segment: A = [40.740, -74.030], B = [40.741, -74.030]
// projPt for a centerline sign at [40.7405, -74.030] is itself.
// Right-perp of this segment: dY>0, dX=0 тЖТ perpX=1 (east), perpY=0
// dir=1 тЖТ pin shifts east (dLng > 0); dir=-1 тЖТ west (dLng < 0)
initRoadGeometry({ "BLOOMFIELD ST": [[[40.740, -74.030], [40.741, -74.030]]] });
initStreetParity({ "BLOOMFIELD ST": 1 }); // odd addresses on east (+1) side
```

**Given** a sign `{ address: "101 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }` (odd, `forcedDir=1` тЖТ east),
**When** `renderSignPins([sign], now)` is called,
**Then** `mockMapInstance._layers.filter(l => l._options["pane"] === "towSignPane")[0]._lng > -74.030`.

**Given** a sign `{ address: "100 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }` (even, `forcedDir=-1` тЖТ west),
**When** `renderSignPins([sign], now)` is called,
**Then** `mockMapInstance._layers.filter(l => l._options["pane"] === "towSignPane")[0]._lng < -74.030`.

**Given** `initStreetParity({})` (no entry for BLOOMFIELD ST), sign `{ address: "101 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }`,
**When** `renderSignPins([sign], now)` is called,
**Then** `mockMapInstance._layers.filter(l => l._options["pane"] === "towSignPane")[0]._lng === -74.030` (no offset applied).

**Given** `initStreetParity({ "BLOOMFIELD ST": 1 })` and a sign `{ address: "101 BLOOMFIELD ST", lat: 40.7405, lng: -74.030 }`,
**When** `renderUpcomingSignPins([sign], now)` is called,
**Then** `mockMapInstance._layers.filter(l => l._options["pane"] === "upcomingPane")[0]._lng > -74.030`.
