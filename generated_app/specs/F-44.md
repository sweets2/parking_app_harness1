# F-44 — Fix: Correct Sign Positions Using Address-Number Interpolation

## What this is

The Hoboken permit API geocodes some signs to the wrong position on their street.
For example, "0000729-0000729 PARK AVE" is geocoded to the 9th St × Park Ave
intersection rather than to the building's actual location within the block. The
visible result is that the pin appears at the intersection end of the block when it
should sit a few meters up the block away from the corner.

This feature adds a pre-processing step that uses **address house numbers as the
source of truth** instead of relying on the geocoder. All signs for a street are
examined together: if one sign's geocoded arc position along the road violates the
monotonically increasing order expected from its house number, its lat/lng is
replaced with an interpolated value derived from its correctly-geocoded neighbors.

---

## Module contract

### `app/map.ts`

**Add two new exports** and four private helpers.

#### `export function getRoadGeometry(): RoadGeometry`

Returns the module-level `_roadGeometry` so `main.ts` can pass it to
`correctSignPositions` without the function needing to close over module state.

#### `export function correctSignPositions(signs: Sign[], roadGeometry: RoadGeometry): Sign[]`

Pre-processes a sign array before rendering. Returns a new array with corrected
lat/lng for any sign whose geocoded position violates monotonic house-number
ordering along its street. Signs that are already in correct order are returned
unchanged (same object reference). Streets with fewer than 3 signs are skipped.

**Algorithm:**

1. Group signs by `normalizeToGeometryKey(streetName)`.
2. For each street with ≥3 signs and known road geometry:
   a. Compute `houseNumMidpoint(address)` for each sign.
   b. Compute `snapToArcPos(ways, lat, lng)` — the arc distance (metres) from the
      start of the concatenated ways to the projected sign position.
   c. Sort by house number (ascending).
   d. Count **inversions**: for each pair (i < j by houseNum) where
      `arcPos[i] > arcPos[j] + 10m`, increment both signs' inversion counts.
   e. If exactly one sign has the highest inversion count (> 0), it is the outlier.
   f. Interpolate the outlier's arc position from the two nearest clean neighbors
      (one with lower houseNum, one with higher). Convert back to lat/lng with
      `arcPosToLatLng`.
3. Return corrected array; if no corrections, return the original array unchanged.

**Private helpers (all internal to `app/map.ts`):**

- `houseNumMidpoint(address)` — parses `(start + end) / 2` from
  `"0000729-0000729 PARK AVE"` (supports leading zeros, single addresses).
- `snapToArcPos(ways, lat, lng)` — flattens all ways, finds nearest projected
  point, returns cumulative arc metres from the start.
- `arcPosToLatLng(ways, arcPos)` — walks the flattened ways to the given arc
  position and returns the interpolated `[lat, lng]`.

### `app/main.ts`

Import `correctSignPositions` and `getRoadGeometry` from `./map`.

After all sign fetching is complete and before `createApp`, add:

```typescript
signsData = { ...signsData, signs: correctSignPositions(signsData.signs, getRoadGeometry()) };
```

This runs once, after `road-geometry.json` and `street-parity.json` are loaded.

## Tests

Tests in `tests/unit/map.test.ts` under `describe("F-44 correctSignPositions")`.

**Given** 3 signs on a N-S road where house 200 is geocoded too far north (past house 300),
**When** `correctSignPositions` is called,
**Then** the corrected lat for house 200 is between house 100's lat and house 300's lat.

**Given** 3 signs already in monotonic order,
**When** `correctSignPositions` is called,
**Then** all signs are returned with their original coordinates unchanged.

**Given** only 2 signs on a street (insufficient data),
**When** `correctSignPositions` is called,
**Then** signs are returned unchanged.

**Given** signs on a street not present in road geometry,
**When** `correctSignPositions` is called with an empty geometry object,
**Then** signs are returned as-is.
